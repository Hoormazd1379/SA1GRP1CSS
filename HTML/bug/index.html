BUGS AND ERRORS

A short definition

A bug in informatics, is an error in a software that can cause several problems in the execution of the software itself. A software with bugs is defined buggy. There are different types of bugs that can cause different types of problems, which the severity may be high depending on quantity and severity of the bugs. Generally, a bug can be caused mainly by three circumstances: syntax error, logic error and runtime error.

The main source of the bugs is usually the programmers who develop the software, but that’s not always the case. It can happen that the compiler doesn’t produce the correct code. In order to solve these bugs, programmers search them to solve them otherwise programmers can also implement a software that can detect and correct bugs. The act to find and solve bugs is called debugging. Is important to check for bugs in a software because the damage that can produce can be enormous, especially if we consider financial damage.
