Debugging

What is Debugging

The identification of a bug by the program or programmer is called Debugging. Its activity is one of the most fundamental operations for the development of a program, which is usually extremely difficult due to the complexity of the software that is currently in use and delicate due to the possibility of introducing new errors or different behaviors from those desired in an attempt to correct those for which the debugging activity took place.

Although each debugging session is unique in its kind, there are some generic principles, according to which they are applicable in every situation. To begin with we can find five steps: identification of the bugs; identification of the component containing the bug; identification of the cause of the bug; design of a bug fix and implementation and test of the above correction. However, there are two ways to perform a debug session..

Debugging Sessions

To begin, you can insert in the most appropriate points of the instructions of the source code of the program which print on the video the contents of the variables that could have the wrong values which consequently could lead to the error. Typically the conditions that lead to an error could arise long before the time the error occurs. We can consider for example the division of two numbers, where if a numeric variable x that will be used later as a divisor of a number contains a value that will make the divisor zero, this will surely cause an error if the value of x remains such:

1/(x-z)

to "debug" this part of the program it is possible to convince

oneself by printing on screen that x-z is zero before the error occurs:

int x=1, z=1:

...

System.err.println("x-z=",x-z);

...

1\x-z

Instead the second way is to execute the program under the control of an application called debugger that allows to execute the instructions of a program one at a time, they allow to carry out the program up to the attainment of determined points (points of interruption) definable by the user and allows you to analyze the contents of the variables at any time. Debuggers play a very useful role in discovering the causes of errors.

Once the causes of the error have been identified and corrected, you can remove the printing instructions to end the debugging session.
