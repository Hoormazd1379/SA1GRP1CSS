Vulnerabilities

Vulnerability is a weakness of a system which can be exploited by a threat actor to perform unauthorized actions within the system. Some vulnerabilities are directly tied to bugs in the code, some are an innate repercussion of the type of the software, and others are the result of a lack of implementation of mitigation strategies. What they all have in common is that they are not a security hazard by themselves, but only when a third party exploits them.

Malicious Code

Malware - short for malicious software - is a type of software made to cause harm, such as by stealing data, gaining unauthorised access to networks or damaging devices. Unlike vulnerabilities and bugs, malware is made by people to intentionally cause damage, and can exploit both bugs and vulnerabilities for that purpose. It is therefore important to know the different types of malware, ranging from viruses, trojans and worms to adware, key loggers and many others, and also how to protect yourself from them.

Bugs and Errors

A bug in informatics, is an error in a software that can cause several problems in the execution of the software itself. A software with bugs is defined as buggy. Different types of bugs exist which can cause various types of problems, which can lead have consequences ranging from insignificant to severe depending on quantity and criticalness of the bugs. Generally, a bug can result from three circumstances: syntax errors, logic errors and runtime errors.
