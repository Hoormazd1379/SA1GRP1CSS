Vulnerabilities

Uncontrolled Resource Consumption

Statistics

  • Type: Class

  • CWE Number: 400

  • Applicable Platforms: Language-independent

  • Likelyhood of exploit: High

Description

Uncontrolled resources consumption is a type of vulnerability that can occur if the size of limited resources is not controlled. (14)
What is limited resources?
Limited resource or System resource is a device that is connected to a computer, where it is interal or external. It can be memory, CPU, Network connections and so on

If the number of resources is not limited, then in the case of any attack, in which this resource will be allocated, the Server Resource manager can crash, consequently making software slower.(13)

Consequences

The most common result of resource exhaustion is denial of service. The software may slow down, crash due to unhandled errors, or lock out legitimate users.(15)

Sometimes the attacker is able to “Fail Open” the software.

Detection

Certain dynamic analysis techniques are able to identify this vulnerability by sending many requests in a short time span.

Fuzzing can also sometimes spot resource consumption, although it is not normally used for this purpose.

Mitigation

In order to avoid problems one needs to put a limit to the amount of processed resources, which in turn will not let attackers allocate uncontrolled amounts of resources. Consequently the software will be less vulnerable.
