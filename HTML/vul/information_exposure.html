Vulnerabilities

Information Exposure

Statistics

  • Type: Class

  • CWE Number: 200

  • Applicable Platforms: Language-independent

  • Likelyhood of exploit: High

Overview

Information exposure is the intentional or unintentional leak of information to someone that is not recognised to have access to that information.

This information can be sensitive within the product functionality, i.e. a private message that has been sent. It can also be in the form of Information that provides material about the product which can then be used in an attack, such as an installation path.(26)

Variants

Information Exposure occurs through numerous different ways.
For Example:

  • Through Sent Data

    The accidental exposure of sensitive information through sent data refers to the transmission of data which are either sensitive in and of itself or useful in the further exploitation of the system through standard data channels.(27)

  • Through Discrepancy

    The product behaves differently or sends different responses in a way that exposes security-relevant information about the state of the product, such as whether a particular operation was successful or not.(28)

  • Through An Error Message

    The software generates an error message that includes sensitive information about its environment, users, or associated data.(29)

  • Through Debug Information

    The application contains debugging code that can expose sensitive information to untrusted parties.(30)

  • Through Caching

    The application uses a cache to maintain a pool of objects, threads, connections, pages, or passwords to minimize the time it takes to access them. If implemented improperly, these caches can allow access to unauthorized information.(31)

Consequences

Loss of confidentiality is the most common consequence, as data is often disclosed. This often has security implications.

For example, if an attacker attempts to retrieve an object from a database, and an SQLException is raised whilst querying the database, an error message will be created. The error message can include sensitive information about the database or query logic. This data can then be used to simplify different attacks, such as an SQL Injection in order to access the database.

Detection

Regular error-checking methods will identify this vulnerability: both dynamic analysis and static analysis.

Scanning the software with Dynamic Analysis through the use of: Web Application Scanners, Web Services Scanners and Database Scanners are highly cost-effective methods which can be used to detect areas which have the potential to cause information to be exposed.

Another highly cost effective method to detect areas viable to information exposure is Manual Static Analysis via the source code, which is achieved through manual code reviews.

Mitigation

In order to prevent such attacks, effective security measures are required, for example:

  • Identifying data that requires extra protection, and protecting it through data encryption, makes it harder for attackers to determine the information that has been exposed.

  • Sites should ensure that there are appropriate section components which are built into the system design and that the compartmentalization allows for and further reinforcing privileges in separation functionality, allowing different levels of privileges.
