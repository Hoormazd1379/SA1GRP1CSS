Vulnerabilities

Improper Input Validation

Statistics

  • Type: Class

  • CWE Number: 20

  • Applicable Platforms: Lanuguage-independent

  • Likelyhood of exploit: High

Description

Improper Input Validation is one of the most common vulnerabilities that can create fatal failure in software. So what is Input Validation?
In software, in order to avoid any kinds of data flow one needs to validate input correctly. So, when one has improper input validation the software has a hard time checking the inputs, which in turn makes the software vulnerable. The Validation of data is very essential, because in software validation of forms reduces the chance of hack attacks and also avoids unintential software crashes. So, if data is validated improperly it can lead to one of these and cause a software failure.(17) There are several reasons for improper input validation. These are:

  1. Shared names of Forms
  2. Improper method of validation
  3. Validator issues
  4. Outdated Logic of validation
  5. Action Form
  6. Disabled Validator

Shared Names of Forms

This problem occurs when two or more forms have the same name, which leads to increased susceptibility to attacks of some forms. Why? Because the Validator only validates one of the forms with the same name and ignores others, which makes other forms more vulnerable.

Improper method of validation

Some classes in a software may not support forms of validation and in order to avoid problems, super validate form should be used. If super validate forms is not used then some classes will not be validated and will be more susceptibile to attacks and software will be more vulnerable

Validator

In the validation process, it is essential to all forms to be validated by forms defined by the Validator, otherwise forms which do not support the Validator will be weak places of a software. Moreover, a possible problem may occur when not all of the fields are validated, because the Validator requires all fields to be validated, even the unused fields as well.

Outdated Logic of validation

The logic of validation should always be updated. There are several consequencese when validation is out of date. One of them are forms with no fields, which may lead to a software crash. Moreover, when validation logic is outdated that means that validation is unavailabe, which makes forms more susceptibile to attacks and software becomes more vulnerable. So, it is essential for programmers to update validation logic.

Action Form

Action forms have their own form defined by "Action Form Mapping". So, this form is the supported by Validator then Action forms will stay unvalidated, which will make tje program more vulnerable.

Disabled Validator

This problem occurs when the Validator is turned off. That can happen when action form disables it. It is very dangerous if the Validator is disabled, because the Validator validates all the forms in a software.

Consequences

The attacker is sometimes able to read the memory, causing loss of confidentiality and leading to data leaks, or even write to memory leading to loss of integrity.

Another possible consequence is denial of service.

Detection

Scanning the software with dynamic analysis and fuzzing are both highly cost-efficient ways of checking for improper input validation, even if manual spotchecks and manual code reviews reviews are often enough to find an anomaly.

Mitigation

There are several mitigations of Improper Input Validation. One of them is to use Strut or OWASP ESAPI Validation API to validate data.
