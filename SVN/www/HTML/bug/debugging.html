Debugging

The debug indicates the activity that consists in the identification by the programmer of the part of software affected by a bug detected following the use of the program. The debugging activity is one of the most important operations for the development of a program, often extremely difficult due to the complexity of the software currently in use and delicate due to the danger of introducing new errors or behaviors that differ from those desired in an attempt to correct those for which the debugging activity took place.

Although each debugging session is unique, there are some generic principles are applicable in every situation. We can recognize five phases: bug identification; identification of the component containing the bug; identification of the cause of the bug; design of a bug fix and implementation and testing of the aforementioned correction. There are two ways to perform a debugging session.

The first possibility is to Insert in appropriate points of the program source code instructions that print on video the contents of variables that could contain incorrect values that lead to the error. Typically, in fact, the conditions that lead to an error could arise long before the time when the error occurs. For example, if a numeric variable x that will be used later as a divisor of a number contains a value that will make the divisor zero, this will certainly lead to an error if the value of x remains that:

if we consider x=1 and z=1; and we consider also the calculus 1/(x-z)

to "debug" this part of the program it is possible to convince

oneself by printing on screen that x-z is zero before the error occurs:

int x=1, z=1:

...

System.err.println("x-z=",x-z);

...

1\x-z

Whereas the second way is to run the program under the control of an application called a debugger. A debugger allows you to execute the instructions of a program one at a time, allows you to run the program until certain user-definable points (breakpoints) are reached, and allows you to examine the contents of the variables at any time. Debuggers are very useful tools to find out the causes of errors.

Once the causes of the error have been identified and corrected, the printing instructions can be removed to end the debugging session.
