Crash

Introduction

The crash indicates the sudden closure or block, of which it was not requested, of a running program or the complete block of the computer. This can happen when there are design or programming errors in the software or due to a malfunction of the hardware that performs it (for example due to excessive overheating). This means that computers crash abnormally due to some errors in the operating system software or errors in computer hardware, but computer hardware errors are, despite being less common, more devastating and more difficult to identify with respect to errors in the software, in fact, a variety of hardware components must work properly for a computer to work and over time these components may not even work.

In this page we can see how to crash a determinate electronic component, which we start with the Operating System, then we continue with the Hardware and at the end we finish with the Software. In the first electronic component, we will see what happen when an operating system, in especially we'll talk in general consider Windows, Mac OS, iOS (for Apple mobile devices) and Android crash. Then in the hardware we can find the RAM and the disk computer, to conclude this section we talk about Software, in which there are programs.
