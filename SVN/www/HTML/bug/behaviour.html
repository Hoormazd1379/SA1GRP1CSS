Behaviour

Effects on the program behaviour

If a program runs and doesn't crash it does not mean it hasn't any bugs, it could have some of them. Since the program is working, we expect it to behave in the way we intended to and to spit out the right results. Unfortunately not all that glitters is gold, which means, although the program is running, a bug present in the code can make the program give results that differ from the expected ones. It could have many causes, for example there can be an arithmetic overflow somewhere in a calculation, or a syntax error in a formula where rather than adding (+) it multiplies (*).

It can also happen that the keyboard input for a platform game is not defined correctly and the game thinks you pressed the right arrow, when you are actually pressing the left one.
