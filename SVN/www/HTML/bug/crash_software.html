Software Crash

A program crash happens when a fatal error has occurred within the application software. This type of crash might result in one specific application not being usable, or it could result in the entire computer not being usable. It's possible for one application to cause a complete computer crash. While operating systems try to isolate the problems to a specific application, so the computer doesn't crash, it can sometimes happen.

Program crashes happen for a number of reasons, and it's often not advised to go on using the program in question until the error is corrected. Most operating systems for desktops, laptops and smartphones have a means by which you can shut down and stop a particular program or app. If the operating system is still functioning, then it's usually possible to shut down the specific program and restart it to continue using the program.

Then we take up the open discourse in hardware and we hypothesize to have a program in which it considers a number between 1 and 9 to be entered in it, but we insert a name. So there is a question: "What happens?". The program tries to use that string of characters as a number, but it doesn't go so well in fact one of the two situations manifests itself at this point: your program continues to work but does the wrong thing: therefore it is up to us programmers to understand that the output coming out is not what we expect. Despite this it is important to understand that the system continues to work without stopping.

The program stops abnormally with the operating system that sends an alert. This is called "exception" or "error". To undertand the second modality we have to examine briefly the architecture of the system. The application is performed in a box created by the hardware (CPU) and the operating system so that it is protected from bugs. The role of the CPU in this context is to check if the application is doing something it shouldn't do (for example, to go beyond its limits): if it happens, the CPU interrupts the execution of the application and calls the code to the rescue in the operating system to protest. This code checks what happened and displays the crash message. When a program has bugs, sooner or later it will execute random or incorrect instructions.

But there is a good news: the programs are separate enough that they can't take the computer with them when they crash, but as always there are exceptions.
