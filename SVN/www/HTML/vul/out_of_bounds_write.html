Vulnerabilities

Out of Bounds Write

Statistics

  • Type: Base

  • CWE Number: 787

  • Applicable Platforms: Most prevalent in languages C/C++/Assembly

Description

Out of Bounds write is a type of vulnerability that happen when a software writes data outside of the current buffer (where physical memory is stored) where it was initialy planned to.

This vulnerability mostly occurs in Assembly, C and C++.(23)

Variants

There are two main Variant type vulnerabilities of out of bounds write: Stack-based buffer overflow (CWE 121) and Heap-based buffer overflow (CWE 122).

Stack Overflow

Stack-based buffer overflow (or stack overflow) is a condition where the buffer being overwritten is allocated on the stack (i.e., is a local variable or, rarely, a parameter to a function).

This vulnerability is not to be confused with stack exhaustion, usually a result from an excessively recursive function call.(24)

Heap Overflow

Heap-based buffer overflow (or heap overflow) is a condition where the buffer that can be overwritten is allocated in the heap portion of memory, generally meaning that the buffer was allocated using a routine such as malloc().(25)

Consequences

Write out of bounds can lead to several problems like troubles in code launching and errors in data, such as the modification of memory (loss of integrity).

Moreover, a program may refer to memory locations which are not in the boundaries of current buffer and that may lead to software failure (loss of aviability).

Detection

This weakness can often be detected using automated static analysis tools. Many modern tools use data flow analysis or constraint-based techniques to minimize the number of false positives.

It may also prove useful to use fuzzing or other dynamic tools.

Mitigation

There are several ways to mitigate this problem. However, most of them only may help partially, but there is a universal solution.
It is the usage of languages, such as Java, that do not support such vulnerability.
