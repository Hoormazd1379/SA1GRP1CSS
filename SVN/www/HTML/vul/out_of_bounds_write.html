Description

Out of Bounds write is a type of vulnerability that may happen in a software when it writes data outside of the current buffer (where physical memory is stored) it was initialy planned to.

Where this can happen

This vulnerability mostly occurs in Assembly, C or C++,

Consequences

Write out of bounds can lead to several problems such as: troubles in code launching, system failure, and errors in data. Moreover, a program may refer to memory locations, which are not in the boundaries of current buffer and may lead to software failure.

Detection

Mitigation

There are several ways to mitigate this problem. However, most of them only may help partially, but there is a universal solution.
It is the usage of languages,such as Java that does not support such vulnerability.
