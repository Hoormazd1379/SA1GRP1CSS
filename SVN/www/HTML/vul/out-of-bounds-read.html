Out of Bounds Read

Description

Out of Bounds read is type of vulnerability that may happen in a software when it reads data from the buffer (place where physical memory is stored) and reads the data out of its bounds.

Potention causes

There are several causes that can trigger Buffer over-read. These are:

  • Ordinary Bugs
  • Malicious Inputs
  • Heartbleed bug(Caused by Improper Input Validation)

Where this can happen

This can mainly happen in C or C++, because function in these languages does not check memory bounds they use. Moreover, it is crucial to know some function, which are bounded in memory use by mistake can cause Buffer over read.

Consequences

When there esists an Out of bound vulnerability then it can lead to the leak of very important information, for intance passwords. Moreover it can lead to the crash of.
Why?
Because software can read so much data that program can cause buffer over-flow, which leads to system crash.
