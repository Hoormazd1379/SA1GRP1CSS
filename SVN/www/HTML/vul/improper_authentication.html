Vulnerabilities

Improper Authentication

Statistics

  • Type: Base

  • CWE Number: 89

  • Applicable Platforms: Database server

  • Likelyhood of exploit: High

Overview

Improper Authentication takes place when an application incorrectly validates the identity of a user, this can cause an attacker to receive certain privileges which can be used for example to have access to sensitive information, which they would otherwise not have.

The intention of the attacker is usually to gain access to an account and be able to have the same privileges as the user. (34)

Relationships

Improper Authentication can be the result of an SQL Injection/SQL Injection related vulnerabilities.

Consequences

Improper Authentication can have different consequences depending on what the intentions of the attacker are and what privileges they have gained. It can lead to a loss of confidentiality, through the attacker gaining access to disclosed information.

They can also, if privileges allow, execute arbitrary code which will be harmful to the application and cause alterations.

Detection

Manual Static Analysis is an effective method used to detect the weakness, this involves manual analysis such as penetration testing, threat modelling. The benefit of such method is that it is useful for evaluating the correctness of custom authentication mechanisms.

Other methods include:

  • Dynamic analysis with Manual Results Interpretation

  • Automated Static analysis

  • Architecture or Design Review

Mitigation

In order to protect applications and websites from Improper Authentication, secure authentication methods that feature anti brute force and session protection mechanisms must be applied. This can be done using an authentication framework or library such as the OWASP ESAPI Authentication which is a free, open source security control library.
