Vulnerabilities

Out Of Bounds Read

Statistics

  • Type: Base

  • CWE Number: 125

  • Applicable Platforms: Most prevalent in languages C/C++

Description

Out of Bounds Read is a type of vulnerability that may happen in a software when it reads data from the buffer (where physical memory is stored) and reads the data out of its bounds.

This can mainly happen in C or C++, because functions in these languages do not check the memory bounds they use. Moreover, it is crucial to know some functions, which are bounded in memory use by mistake can cause buffer over read or buffer under read.(18)(19)

Variants

Buffer over-read (CWE 126) is a Variant type vulnerability categorized under out of bounds read. It usually happens when a pointer is incremented over the limits of the buffer.(20)

Buffer under-read (CWE 127) is a Variant type vulnerability categorized under out of bounds read. It usually happens when a pointer is decremented under the limits of the buffer.(21)

Potential Causes

There are several causes that can trigger an out of bounds read. These are:

  • Ordinary Bugs
  • Malicious Inputs
  • Heartbleed bug (Caused by Improper Input Validation)(22)

Consequences

When there exists an out of bounds vulnerability, it can lead to leakage of very important information, for instance passwords (loss of confidentiality).

Moreover, it can lead to the crash of the program, because the software reads too much data which causes a buffer over-flow, which in turns causes data corruption.

Detection

This weakness can be detected by a standard manual code review, for example by checking in there exists a check within bounds for all pointers and values.

Mitigation

There are 2 ways to mitigate this problem.

  • The first is to use a language with restricted memory access.
  • The second is to use input validation. Moreover when using Input validation it is essetial to be aware about its potential problems, which are here : Improper Input Validation.
