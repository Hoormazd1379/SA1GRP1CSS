Vulnerabilities

Improper Restriction Of Operation Within the Bounds of Memory Access

Statistics

  • Type: Base

  • CWE Number: 89

  • Applicable Platforms: Database server

  • Likelyhood of exploit: High

Overview

Various languages can allow direct addressing of memory locations and do not automatically ensure that these locations are valid for the memory buffer that is being referenced. This can cause read or write operations to be performed on memory locations that may be associated with other variables, data structures, or internal program data.

Simply, a software can execute operations on the memory buffer, but when there are improper restrictions, it can cause it to read from or write to a memory location which is not located inside of the intended memory buffer.

Consequences

Confidentiality: In the event of an out of bounds read, the attacker may gain access to sensitive information. The sensitive information may contain system details, for example, the current buffers position in memory, this can be used to orchestrate further attacks, possibly with more severe consequences.

Out of bounds memory access has a high chance of causing corruption of relevant memory, and possibly instructions, which in turn, can cause the system to crash.

If the memory that is accessed by the attacker is controlled, it allows the possibility to execute arbitrary code, as with a standard buffer overflow.

Detection

The most effective way of detecting the weakness is through Automated Static Analysis tools, an IEEE inspection can also locate the weakness but it is a much more costly method.

Mitigation

Defending applications from buffer errors can be done by performing checks on input data, and through the use of certain compilers which provide process that can mitigate or eliminate exploitation possibilities, such as automatic buffer overflow detection.

Language Selection:
Using a language such as Java and Perl do not allow this weakness to occur, this is excuse they perform their own memory management, thus they are not liable to buffer overflows. Ada and C# are similar in that they provide their own overflow protection.
