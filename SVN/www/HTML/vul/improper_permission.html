Vulnerabilities

Incorrect Permission Assignment For Critical Resource

Statistics

  • Type: Base

  • CWE Number: 89

  • Applicable Platforms: Database server

  • Likelyhood of exploit: High

Overview

Incorrect Permission Assignment for Critical Resource is when a certain resource has permissions that allow access by more actors than required. This can lead to the exposure of sensitive information, or alteration of the resource if they have write permissions. This is a severe weakness when the resource contains information related to program configuration, execution or sensitive user data.

In short, the software specifies permissions for a security-critical resource in a way that allows that resource to be read or modified by unintended actors.

Consequences

Any application that is writable by an unintended party compromises the system security, for example, they would be able to destroy or corrupt critical data in the targeted resource, such as deletion of records from a database.

The attacker may have the means to read sensitive information from the resource, such as credentials, configuration information or user data, resulting in a loss of confidentiality

If the attacker has access to a world-writable executable, they are able to inject malicious content into it and execute arbitrary code, such as a Trojan horse.

Detection

A highly cost effective method to detect flaws in permission assignments is Manual Static Analysis via the source code, which is achieved through manual code reviews.

Automated Static Analysis can be effective in detecting flaws in permission assignments for system resources such as files, directories, shared memory, device interfaces, etc. Automated techniques may be able to detect the use of library functions that modify permissions, then analyze function calls for arguments that contain potentially insecure values.

Mitigation

Make sure that all configuration files, executables and libraries can be readable and writable by only the administrator.

During program startup, set the default permissions to the most restrictive assignments possible.

When using a critical resource such as a configuration file, check if the resource has insecure permissions, generate an error or exit the software if there is a possibility that the resource could have been modified by an unauthorized party, as there may be a chance it contains malicious code.
